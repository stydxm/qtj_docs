# 容器 & 进程

## 概念
介绍容器之前，必须回顾ROS的节点间是什么关系
<!-- 参考前置课程 -->
我一般来说我们希望一个节点实现一个功能，这样从功能的角度上实现了代码解耦，互相之间不会调用代码，编译时也能提升开发者效率

<!-- + 问什么要有容器 -->
每换一种节点组合，都要重新编写和编译程序，很不方便   
容器就是ROS2提供的通用解决方案。它作为整个进程唯一的主程序，负责加载和运行其他节点   
```bash
容器进程
  ├── 相机节点
  ├── 图像识别节点
  └── 目标跟踪节点
```

由main()带起来可执行程序，就会产生一个进程，而进程间的通讯往往是低效的。[参考](/https://discourse.openrobotics.org/t/impact-of-ros-2-node-composition-in-robotic-systems-ra-l/31474/2)
<!-- 代价是什么 -->

那这么好用的东西为啥放在后面讲？   
*单进程风险* : 一个组件的故障就要整个进程重启，有时得不偿失(比如串口比特错位，你想等整个视觉模块加载么)
